# Security Policy

## Supported Versions

Actualmente este proyecto no mantiene múltiples versiones liberadas.
Las correcciones de seguridad se aplican únicamente a la rama activa de desarrollo.

| Version | Supported |
| ------- | --------- |
| `main` (última versión) | :white_check_mark: |
| Ramas antiguas / forks desactualizados | :x: |

## Reporting a Vulnerability

Si encuentras una vulnerabilidad de seguridad, por favor **no la publiques en issues, discusiones o pull requests públicas**.

Canales recomendados:

1. Reporte privado en GitHub (Security Advisories, si está habilitado en este repositorio).
2. Correo institucional: `tics@itsoeh.edu.mx` con el asunto: `[SECURITY] Reporte de vulnerabilidad`.

Incluye, cuando sea posible:

- Descripción del problema y posible impacto.
- Pasos para reproducir.
- URL/ruta afectada.
- Evidencia técnica (capturas, logs, PoC mínima).
- Recomendación de mitigación (opcional).

### Tiempos de respuesta

- Acuse de recibido: dentro de 72 horas.
- Evaluación inicial: dentro de 7 días hábiles.
- Actualizaciones de seguimiento: al menos una vez por semana hasta su cierre.

### Proceso de atención

Si el reporte se confirma:

- Se clasifica severidad e impacto.
- Se prepara corrección y despliegue.
- Se coordina la divulgación responsable con la persona reportante.

Si el reporte no aplica:

- Se explica la razón (por ejemplo: comportamiento esperado, riesgo no explotable o falta de evidencia reproducible).

Gracias por ayudar a mantener este proyecto seguro.
